PyLint

  • It is a static code analysis tool used to check for errors and enforces good coding standards
  • Some types of check include:
    • Checking the length of each line
    • Checking that variable names are well-formed according to the project's coding standard
    • Checking that declared interfaces are truly implemented.
  • The following steps can be followed to install pylint:
    • In the terminal type the command - pip install pylint
    • To run the test, use the command - pylint filename.py
Here is a simple program (sample.py) having some styling issues:
a = 23
b = 45
c = a + b
print(c)

OOPs Glossary



Factory&Instantiation

Factory: A factory is an object for creating other objects. It is a function or method that returns objects of a varying prototype or class from some method call, which is assumed to be “new”

Instantiation: Is the process of taking a class and creating an object from it.

Class&Object

Class: Is a blueprint or set of instructions to build a specific type of object.
Syntax:
Class class_name{
Field;
Method;
}

Object: Is an instance of a class. It is a self-contained component which consists of methods and properties to make a particular type of data useful.

Namespace&Constructor

Namespace: Is a declarative region that provides a scope to the identifiers inside it.

Constructor: It is a member function of a class which initializes objects of a class.

Fixture&UnitTest

Fixture: These are required to arrange or set up the test environment for test functions.

Unit test: Is a testing method by which individual units of source code are put under various tests to determine whether they are fit for use.

Type Hint&Cast

Type hint: Is a formal solution to statically indicate the type of value within code.

Type cast: Is a method used to change the variables or values declared in a certain data type into a different data type to match the operation required to be performed by the code.

Static&Class Methods

Static: Is a method which cannot access or modify the class state.

Class: Is a function decorator that is an expression that gets evaluated after the function is defined.

Facade&Design Patterns

Facade: Is a structural design pattern that provides simpler unified interface to a more complex system.

Design patterns: They provide general repeatable solution to a commonly occurring problem.

Method&Data Member

Method: Is a function for a given object.

Data member: A class variable or instance variable that holds data associated with a class and its objects.

Method&Operator Overloading

Method overloading: The assignment of more than one behavior to a particular function.

Operator overloading: Assignment of more than one function to a particular operator.

Class&Instance Variable

Class variable: A variable that is shared by all instances of a class.

Instance variable: A variable that is defined inside a method and belongs only to the current instance of a class.
